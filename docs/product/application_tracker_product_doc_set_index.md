# Application Tracker Product Document Set

This document set reframes Application Tracker as a goal-aware job-seeking workspace rather than only a workflow board.

## Documents

### 1. Strategic vision
`application_tracker_strategic_vision.md`

Defines the product purpose, thesis, vision, strategic principles, pillars, and success criteria.

### 2. Product analysis refresh
`application_tracker_analysis_refresh.md`

Expands the product analysis to include user intent, AI guidance placement, artefact intelligence, external workflow transitions, and differentiated intake paths.

### 3. Design brief refresh
`application_tracker_design_brief_refresh.md`

Refreshes the design direction so that the product supports both workflow momentum and real application execution work.

### 4. Composite product vision
`application_tracker_composite_product_vision.md`

Combines the near-term and longer-term product framing, user modes, tenets, is/is not boundaries, intake model, email direction, and research notes into one concise narrative artifact.

### 5. Inbox and quiet monitoring decision memo
`application_tracker_inbox_monitoring_decision_memo.md`

Captures the current recommendation for dedicated career inbox support, quiet monitoring modes, user-owned versus centralised email handling, and what must be proven before taking on heavier communication architecture.

## Shared framing

Across the current product docs, the product is framed as:

- goal-aware
- AI-supported
- artefact-informed
- workflow-efficient
- low-friction across internal and external tools
- privacy- and user-ownership conscious

## Core strategic shift

The main shift is from:

**job tracker with board-centric workflow**

to:

**job-seeking operating environment that helps a user move from intent to next role, and potentially over time supports longer-term career opportunity cycles**
